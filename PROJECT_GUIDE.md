# PROJECT GUIDE — Banana Black Website

> 이 문서는 바나나블랙 홈페이지 프로젝트의 전체 기준 문서입니다.
> 코드 작업 전, 방향이 흔들릴 때, 다음 작업을 시작할 때 이 문서를 먼저 확인하세요.

## 1. 프로젝트 개요

| 항목 | 내용 |
|---|---|
| 프로젝트명 | Banana Black Website |
| 목적 | 촬영 문의 전환이 되는 브랜드 포트폴리오 사이트 |
| 운영 주체 | 바나나블랙 (대한민국 기반 커머셜 스튜디오) |
| 기술 스택 | Vanilla HTML / CSS / JS (프레임워크 없음) |

### 핵심 목표 4가지

1. **포트폴리오 신뢰감** — 작업물이 설득력 있게 보이는 것
2. **촬영 문의 전환** — 방문자가 카카오 또는 폼으로 연결되는 것
3. **카테고리 정리** — 제품 / 푸드 / 디저트 / 공간 / 인물을 명확하게 나누는 것
4. **채널 기반 구축** — 인스타 광고 / 크몽 / 숨고 유입을 받을 수 있는 랜딩 역할

## 3. 페이지별 역할

### 홈 (index.html)
- 브랜드 첫인상을 결정하는 페이지
- 히어로 영역 → 브랜드 스테이트먼트 → 카테고리 쇼케이스 → 문의 CTA 순서
- 방문자가 포트폴리오를 보고 싶게 만들거나, 바로 문의하게 만드는 두 흐름을 동시에 지원

### About (about.html)
- 브랜드 톤과 철학을 전달
- 길게 설명하지 않고, 짧은 문장으로 포지셔닝
- 신뢰를 주는 역할이지, 많은 정보를 주는 역할이 아님

### Contact (contact.html)
- 실제 문의 전환이 일어나는 가장 중요한 페이지
- 카카오 CTA → 정식 문의 폼 → 개인정보 동의 구조
- 예쁜 페이지보다 문의 잘 되는 페이지가 목표

### Works Hub (works/index.html)
- 5개 카테고리(Product / Food / Dessert / Space / Portrait)의 인덱스
- 대표 이미지 + 카테고리명으로 빠른 탐색 유도

### 카테고리 상세 (works/product, food, dessert, space, portrait)
- 각 카테고리의 실제 포트폴리오 이미지를 보여주는 페이지
- 이미지가 페이지의 그릇, 텍스트는 최소화
- 라이트박스 → 이전/다음 탐색 지원

## 4. 디자인 방향

### 전체 무드
- 다크톤 기반 (홈 / Works / About / Contact 전 페이지 어두운 배경)
- 미니멀 + 프리미엄 + 상업 포트폴리오 무드
- **이미지가 주역, UI는 조연**

### 핵심 원칙
- 읽히는 고급스러움 — 차갑지 않고 따뜻하지도 않은 중립
- CTA는 조용하지만 약하지 않게
- 여백은 낭비가 아니라 리듬

## 5. 공통 디자인 시스템

### 타이포 위계
| 레벨 | 용도 | 예시 |
|---|---|---|
| 영문 라벨 | 섹션 구분자 | `ABOUT`, `PROJECT INQUIRY` |
| Serif 제목 | 핵심 헤드카피 | `Sharp but Clear.` |
| 한글 제목 | 메인 메시지 | `본질을 더 선명하게` |
| 서브카피 | 보조 설명 | `rgba(230,234,232,0.94)` 톤 (다크 배경 기준) |
| 본문 | 상세 정보 | 소개 텍스트, 폼 설명 |
| 필드 라벨 | 폼 입력 안내 | `브랜드명 / 업체명` |

### 세로 간격 원칙
- CSS 변수 시스템 기반: `--space-sm(1rem)` / `--space-md(2rem)` / `--space-lg(4rem)` / `--space-xl(8rem)`
- 임의의 픽셀값 추가 금지 — 변수 조합으로 해결
- 섹션 간격은 `var(--space-xl)` 기준, 내부 요소는 `--space-md` 이하

### 버튼 시스템
- `.btn-contact.btn-cta-primary` — 다크 다이아몬드 아웃라인 (홈 / About / Works CTA 1순위)
- `.btn-contact.btn-cta-secondary` — 더 약한 아웃라인 (서브 액션)
- `.btn-submit.btn-cta-primary` — 폼 제출 버튼 (Contact)
- 히어로 안 버튼: `.hero-actions` 안에서만 다크 tint 배경 추가 오버라이드
- hover 시: 웜 크림 배경(`rgba(242,238,228,0.94)`) 채우기 + 다크 텍스트

### 고정 카카오 CTA (floating)
- 위치: 화면 **우하단** `bottom: 2rem, right: 2rem` (모바일: 1.5rem/1rem)
- 스타일: 다크 차콜 배경 + 앰버 아이콘(`rgba(194,158,82,0.9)`) + 앰버 테두리 힌트
- 전 페이지 공통 (header 밖 독립 요소)

### 다크톤 대비 원칙
- 메인 텍스트(헤드라인): `#eef0ed` 계열
- 서브카피(본문): `rgba(230, 234, 232, 0.94)` 계열
- 보조 텍스트(라벨, 주석): `rgba(214, 219, 217, 0.8)` 계열
- 구분선: `rgba(255, 255, 255, 0.07~0.13)` 계열
- 완전 흰색(`#ffffff`) 직접 사용 금지

### 핵심 원칙 한 줄
> **"같은 시스템, 페이지별 강도 조절"** — 다크 배경 색조가 페이지마다 약간씩 다르지만, 구조와 간격은 동일하게 유지

## 5-1. 홈 카테고리 레이아웃 (Home 전용)

- Product 카드: 전체 컨테이너 폭 와이드 쇼케이스 (aspect-ratio: 16/8.4)
- Food / Dessert / Space / Portrait: **2×2 균등 그리드**, 모두 `aspect-ratio: 4/5`
- 카드 간격: `gap: var(--space-lg) var(--space-md)` — 행/열 리듬 통일
- 이전 비대칭 스태거 레이아웃은 제거됨 (12열 grid-column 오프셋 → `1fr 1fr` 2열)
- `.category-grid` gap: `var(--space-lg)` (Product↔2열 이미지 간격)

## 6. 카테고리별 레이아웃 원칙

### 이미지 밀도 원칙 (전 카테고리 공통)
- 히어로 이미지 최대 높이: `max-height: 88vh` (뷰포트를 가득 채우는 게 기본)
- 블록 간 세로 간격: `--works-sequence-gap: clamp(1rem, 1.8vw, 1.6rem)` — 여백은 리듬이지 빈 공간이 아님
- 보드/페어 내부 아이템 간격: `--works-grid-gap: clamp(0.45rem, 0.8vw, 0.72rem)` — 촘촘하게
- 시퀀스 폭: `--works-sequence-width: min(100%, 1340px)` 기준 (컨테이너 1400px 최대 활용)
- 이 값들은 카테고리별 특성에 따라 소폭 조정할 수 있지만, 더 넓히는 방향으로만

### Product
- 구조: 큰 컷(와이드) → 2×2 보드 → 큰 컷 → 2×2 보드
- `.works-sequence` + `.works-frame` / `.works-board` 시스템 사용
- 이미지 크롭 없이 원본 비율 유지
- 상업 제품 특성상 배경 여백이 있는 사진이 많으므로 간격을 더 줄여 밀도 유지

### Food
- 구조: 큰 컷 → 2×2 → 큰 컷
- 따뜻한 무드 유지, 음식 질감이 살아나도록 밝기 필터 최소화
- 히어로 이미지를 뷰포트 최대한 활용 (88vh)

### Dessert
- 구조: 큰 컷 → 2열 페어 → 큰 컷
- 디저트와 티, 패키지, 재료 컷이 섞여도 한 톤 안에서 읽히도록 정리
- 밝은 피사체가 많아도 배경은 다크톤 리듬 안에서 유지

### Space
- 구조: 2열 쌍 → 큰 컷
- 공간 특유의 공기감은 필요하지만, 검은 여백이 두드러지지 않도록 간격은 타이트하게 유지
- 히어로 컨테이너가 다른 페이지와 달리 별도로 1440px까지 확장됨 (`#space-hero .container`)
- 히어로 폭: `--works-hero-width: min(100%, 1380px)` — 공간샷의 넓은 화각을 살림

### Portrait
- 구조: 히어로 컷(세로 중앙) → 2열 반복 갤러리
- 인물 사진 특성상 세로 비율 이미지 비중 높음
- 히어로 폭: `--works-hero-width: min(100%, 920px)` — 세로컷 특성에 맞춘 너비
- 갤러리 폭: `--works-gallery-width: min(100%, 1280px)` — 그리드 컨테이너도 1400px까지 확장
- portrait 전용 라이트박스 portrait 오리엔테이션 대응

### 공통 규칙
- 이미지는 `object-fit: contain` — 강제 크롭 금지
- 큰 이미지 기준 좌우 정렬 / 작은 이미지는 묶음 단위로 정렬
- 라이트박스: 가로 이미지는 넓게, 세로 이미지는 좁게 자동 대응
- 중간 설명 카피 금지 — 텍스트는 하단 CTA 구간에만

## 7. 인터랙션 원칙

- **hover 딤**: 이미지에 마우스 올리면 밝기 낮춤 + 확대 아이콘 노출
- **라이트박스**: 이미지 클릭 시 전체 화면 뷰어 — 좌우 화살표, ESC, 배경 클릭으로 닫기
- **같은 카테고리 내 이전/다음**: 라이트박스 내에서 같은 페이지 이미지 순서대로 탐색
- **전환**: 느리고 고급스럽게 (`0.6s~0.82s cubic-bezier`)
- **금지**: 튀는 효과, 빠른 바운스, 스크롤 지연 등 과한 애니메이션
- **제거된 요소**: 홈 히어로 하단 스크롤 인디케이터(세로선) — 역할 없이 시각 방해로 판단해 삭제

## 8. 문의 전환 원칙

Contact 페이지는 정보를 전달하는 페이지가 아니라 **행동을 유도하는 페이지**입니다.

### Contact 페이지 구조 (현재 확정)

| 구간 | 역할 | 핵심 요소 |
|---|---|---|
| Intro (상단) | 안내 — 카카오가 메인 채널임을 한 줄로 선언 | h1 "촬영 문의" + 1줄 desc |
| INQUIRY GUIDE | 채널 정보 — 카카오 우선 강조 + Tel/Email 값 직접 노출 | 제목 + 전화번호·이메일 링크 |
| FASTEST CTA | 행동 유도 — 카카오 버튼 full-width + 2줄 안내 | amber border-top, 버튼 전체 폭 |
| FORMAL INQUIRY | 정식 접수 — 폼 + 개인정보 동의 | 1줄 desc + 폼 + full-width 제출 버튼 |

### 운영 원칙
- **카카오 CTA 최우선**: 가장 빠른 전환 채널 — FASTEST CTA 패널 버튼 전체 폭(full-width)
- **중복 정보 금지**: 같은 채널(전화/이메일)이 두 형태로 중복 노출되지 않도록 관리
- **"보조 채널입니다" 문구 금지**: 전화/이메일을 설명하는 설명문 넣지 말 것
- **정식 폼은 보조**: 공식적인 의뢰를 원하는 고객 대상 — 제출 버튼 full-width로 폼 마무리 명확화
- **개인정보 동의 포함**: 법적 최소 요건 충족
- **전화 / 이메일**: INQUIRY GUIDE 패널에서 값(번호/주소) 직접 클릭 연결

### 하단 CTA 공통 시스템 (Home / About / Works)
세 페이지 하단 CTA는 동일한 리듬과 간격 기준을 공유합니다.
- 구조: 라벨 → 메인 카피(1~2줄, serif) → 서브카피(2줄) → 버튼 2개
- 정렬: center
- 간격 기준: 메인↔서브 1rem / 서브↔버튼 2rem

### 전체 CTA 우선순위
1. 우하단 플로팅 카카오 버튼 (전 페이지 항상 노출)
2. 히어로 / 섹션 내 primary 버튼 (카카오 문의)
3. secondary 버튼 (Works 보기 / 정식 문의 남기기)

## 10. 마지막 작업 원칙

> 기능 추가보다 마감 완성도가 먼저입니다.

1. **덜어낼 건 덜어내기** — 미완성인 채로 남은 섹션은 있느니 없는 게 낫다
2. **정렬 / CTA / 정보 위계 우선** — 새 섹션 추가보다 기존 섹션 정리가 점수를 더 올린다
3. **모바일에서 깨지지 않는 것** — 반응형 마감을 실제 기기로 확인
4. **서체 적용은 맨 마지막** — 레이아웃이 완전히 정리된 후 진행
5. **QA 없이 배포 금지** — 링크, 이미지 경로, CTA, 폼 작동 전부 확인 후 배포
